# 墨远 AI 论文简报

🌐 **在线访问**：[moyuan-maker.github.io/moyuan-ai-brief](https://moyuan-maker.github.io/moyuan-ai-brief)

## 这是什么

**墨远 AI 论文简报**是一份面向 AI 行业从业者的每日论文简报，由墨远出品。

arXiv 的 AI 相关分类每天新增 200–400 篇论文。即使是全职研究员，也不可能全部看完。对于产品经理、创业者、独立开发者来说，这个信息量更是完全不可能消化——但这些人同样需要知道行业在发生什么。

不是每个细节，而是：**哪些技术方向在加速，哪些假设被挑战了，哪些新工具可能改变现有的工作方式。**

## 适合谁读

- **AI 创业者** — 预判技术趋势，了解竞争格局变化
- **产品经理** — 知道哪些技术开始成熟，哪些方向值得立项
- **独立开发者** — 跟踪新工具和新方法，判断技术选型
- **投资人 / 行业分析师** — 持续跟踪 AI 领域的技术脉络

不需要有机器学习背景。解读重点放在"这件事意味着什么"，而不是"这篇论文用了什么方法"。

## 功能

- 📋 按日期排列的每日论文列表，点击进入当日详情
- 📄 每篇论文含：中文标题、机构、顶会信息、一句话摘要、从业者解读、arXiv 原文链接
- 📖 方法论页面，公开筛选规则和评分机制
- 📬 邮件订阅入口

## 评分信号

每篇论文通过规则引擎综合 8 类信号自动打分：

| 信号 | 逻辑 |
|------|------|
| 机构背景 | 来自 Google、OpenAI、Meta、清华、Stanford 等顶级机构加分 |
| 社区推荐 | 被 HuggingFace Daily Papers 收录加分 |
| 社区热度 | HF 投票数越高分数越高（分 4 档） |
| 顶会收录 | 被 ICLR、NeurIPS、ICML 等接收加分 |
| 代码可用 | 有开源代码实现加分 |
| 从业者相关性 | 标题含部署、推理优化、Agent 等关键词加分 |
| 学术影响力 | Semantic Scholar 高频引用加分 |
| 开源热度 | 关联仓库在 GitHub Trending 加分 |

## 技术架构

```
用户访问网站
    │
    ▼
前端调用 Kimi API（moonshot-v1-32k）
    │
    ▼
AI 基于编辑规范生成当日 5 期内容
（每期 3–5 篇论文 + 摘要 + 从业者解读）
    │
    ▼
渲染为按日期排列的列表，点击进入详情
```

- **前端**：纯 HTML / CSS / JavaScript，无框架依赖，单文件
- **AI 模型**：Kimi（月之暗面 moonshot-v1-32k）
- **托管**：GitHub Pages，零服务器成本

## 本地运行

无需安装任何依赖。

**第一步：克隆仓库**

```bash
git clone https://github.com/moyuan-maker/moyuan-ai-brief.git
cd moyuan-ai-brief
```

**第二步：填入 Kimi API Key**

用任意文本编辑器打开 `index.html`，找到：

```javascript
const KIMI_API_KEY = '在此填入你的Kimi API Key';
```

替换为你的真实 Key（从 [platform.moonshot.cn](https://platform.moonshot.cn) 获取）。

**第三步：打开网页**

直接用浏览器打开 `index.html` 即可预览。

## 解读原则

- **先讲问题，再讲方案** — 读者先理解"为什么这件事重要"
- **从业者视角** — 重点是"这跟我有什么关系"
- **保持克制** — 不过度解读，不称所有东西为"突破"
- **可验证** — 每篇解读都附 arXiv 链接，可自行查证

## 已知局限

- 自动化筛选有盲区：小众但高价值的工作可能因缺少社区信号被遗漏
- 评分体系偏工程导向，可能低估纯理论贡献
- 解读基于论文摘要而非全文，深度有限，以原论文为准
- 单一时间切面，不回溯更新历史简报
